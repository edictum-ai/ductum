Built Ductum from design docs to first autonomous agent dispatch on 2026-04-04.

Key technical findings:
- Claude Agent SDK needs `permissionMode: 'bypassPermissions'` + `allowDangerouslySkipPermissions: true` for headless operation — default mode requires human approval per tool call
- Only enforce workflow on work tools (Read/Write/Edit/Bash/Glob/Grep) — framework tools (MCP, Agent, ToolSearch) must bypass or the agent can't function
- SDK model names use hyphens not dots: `claude-sonnet-4-6` not `claude-sonnet-4.6`
- Runs should start in `implementing` not `accepted` — agents don't reliably call gate_check to advance from read-only stage
- The @edictum/core workflow is currently just a tool allowlist — real 10-stage enforcement from edictum-demo coding-guard.yaml is copied in but not wired yet

**Why:** These findings came from 6+ iterations of dispatch debugging. Each was a silent failure.

**How to apply:** When wiring the real workflow, remember that evaluate() alone is not enough — must call recordResult() after allowed tool executions for the workflow to track evidence and auto-advance through exit gates.
