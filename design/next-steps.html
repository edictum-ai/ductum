Phase 3 - UI and operator surface

  • P3-A token/theme foundation landed: dark canonical tokens, status tone bridge, motion tokens, and deterministic agent colors.
  • Local ductum start now mints a short-lived browser handoff for loopback auto-open instead of sending the operator to paste a hidden token.
  • Local HTTP handoff cookies now stick on 127.0.0.1; HTTPS/proxy requests still get Secure.
  • Settings now manages a dashboard session instead of a token-entry form; local reconnect sets the HttpOnly cookie without exposing the operator credential to JavaScript.
  • Remote/shared dashboard browser access now has a one-time pairing path: ductum dashboard pair mints a short-lived link, and Settings can exchange a pairing code for an HttpOnly cookie.
  • Settings now translates protected-session failures to Browser session required instead of leaking raw token/Unauthorized backend phrasing into the UI.
  • The dashboard no longer reads ductum.operatorToken for API headers or SSE URLs; same-origin cookies are the browser path.
  • The auth banner now auto-tries local reconnect on a real 401 and falls back to a fresh dashboard-link instruction.
  • Home now starts with an operator Inbox for approvals and needs-attention runs; Today moved below it.
  • RunDetail now has one audited intervention bar for approve, reject, retry, pause, resume, and cancel, plus an explicit agent-select redirect control; UI and CLI both send operator reasons to the same API.
  • Pause/resume is wired through API, CLI, dashboard hooks, and the parity manifest. Resume only works once while the paused task is still active.
  • Redirect is wired through API, CLI, dashboard hooks, and the parity manifest. It stops the active attempt, preserves the task/worktree, assigns the new agent, and returns the task to ready.
  • The operator-action manifest now covers twelve shipped run actions; approve-rebase, pause/resume, redirect, and budget/turn recovery have CLI/dashboard controls.
  • A source-level conformance test now fails if a manifest entry points at a missing Hono route, missing CLI command, or missing dashboard control file/action id.
  • RunDetail now opens on a mixed timeline of activity, evidence, stage transitions, gates, decisions, and operator updates with SSE state and live-follow control.
  • Protected local Codex HTTP MCP now validates the per-run control token for the matching run, so agent MCP calls do not require the operator token.
  • The named UI/UX roadmap items are closed; broader CRUD/action parity can keep improving under Phase 4 DX.

What works better now

  • Crash recovery is durable enough to survive an API kill and classify an expired lease as resumable.
  • Stale owners are fenced out of recovery-critical writes.
  • Terminal runs are not silently overwritten by lineage cleanup.
  • Checkpoint resume keeps the source run's runtime agent when a task was manually started on a different agent than its assignment.
  • Deterministic poison quarantine is now dogfoodable through a packaged mock-agent sample spec.
  • Cost surfaces no longer pretend unknown usage is free.
  • The dashboard has one canonical dark token layer instead of split status color maps.
  • Local dashboard launch now uses the existing one-shot welcome handoff path instead of making token paste the happy path.
  • The fallback UI now says reconnect/session settings rather than centering the operator token.
  • Local reconnect now refreshes the browser session cookie directly and clears stale legacy manual keys from browser storage.
  • Browser pairing links now use ?pair= instead of ?token=, with old links still accepted.
  • Protected Settings now says Browser session required instead of surfacing raw operator-token or Unauthorized messages.
  • The dashboard no longer sends operator credentials from localStorage or puts them in EventSource URLs.
  • Home now opens on the work that needs the operator, including quarantined and frozen runs with shared status tones.
  • RunDetail no longer splits approve/retry/cancel across separate panels; shipped mutating actions now share one reasoned intervention surface and a parity manifest.
  • Operators can now pause an active attempt and resume the paused task from API, CLI, or dashboard without using cancel/retry as a proxy.
  • Operators can now redirect a live attempt to another agent from API, CLI, or dashboard without using unsafe task reassignment while a run is active.
  • The parity manifest no longer pretends the first four dashboard buttons are the whole action surface; approve-rebase, redirect, and budget/turn recovery are first-class controls now.
  • Parity is now checked against source instead of being a hand-maintained table that could drift silently.
  • Operators now have a direct ductum doctor readiness smoke command before dropping into the fuller ductum repair report.
  • RunDetail now leads with a single newest-first timeline instead of making the operator reconcile six raw tabs by hand.
  • Codex MCP URLs now carry only the run-scoped control token when available; the wider operator token is no longer placed in the agent-facing HTTP MCP URL.

Bugs found while dogfooding

  • Fixed: remote/shared dashboard auth had no non-token browser path. It now has one-time pairing; bearer tokens remain for automation.
  • Fixed: Codex HTTP MCP auth returned Operator token required against the protected local API even though dispatch already minted a run-scoped control token.
  • Fixed: checkpoint resume used an inaccessible glm agent after the crashed run used opus.
  • Fixed: ductum start stripped DUCTUM_MOCK_AGENT_CALLS=1, so mock proof servers silently loaded real harnesses.
  • Hard-killing the API can leave child harness processes for cleanup/recovery to handle.
  • Fixed while implementing controls: the added RunDetail tests exceeded the 300 LOC gate, so interaction coverage was split into a focused controls test.
  • Fixed while implementing timeline: activity rows could duplicate the same redacted raw text as both title and detail; the timeline now omits redundant detail.
  • Fixed while replacing token UX: the auth banner showed only because the API was token-protected, even when a cookie session could already be valid; it now appears after a real 401.
  • Fixed while tightening token UX: protected Settings leaked raw Operator token required / Unauthorized backend language; it now shows browser-session copy.
  • Fixed while expanding parity: approve-rebase was CLI/API only, while budget and turn recovery routes were API-only. They now have CLI/dashboard coverage.
  • Fixed while removing token UX: dashboard SSE URLs could append ductum_operator_token from localStorage. EventSource now relies on the same-origin browser session.
  • Fixed while adding pause/resume: a paused run could be resumed repeatedly because the run remains terminal paused; resume now requires the task to still be active.
  • Fixed while adding redirect: plain task reassignment correctly blocks while a run is active, so redirect now has its own route that stops the current attempt before assigning the task to the next agent.
  • Fixed while closing parity: the manifest test only checked static strings, so an API route, CLI command, or dashboard control could be renamed without failing the parity contract.
  • Fixed while adding doctor: older CLI cutover tests still treated doctor as a retired command group; the tests now distinguish the new readiness command from the legacy removed groups.
